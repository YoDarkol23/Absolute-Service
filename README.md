Клиент-серверное консольное приложение для сервиса доставки автомобилей

Общее описание
Проект представляет собой клиент-серверную систему, предназначенную для предоставления информации о доступных автомобилях и расчёта таможенных платежей.
Серверная часть принимает HTTP-запросы, обрабатывает их в многопоточном режиме и возвращает ответы в формате JSON.
Клиентская часть реализует текстовое меню для взаимодействия пользователя с сервером через консоль.

Структура проекта

testCarDelivery/
├── CMakeLists.txt Главный файл сборки
├── common/ Общие компоненты (используются клиентом и сервером)
├── server/ Исходные файлы серверной части
├── client/ Исходные файлы клиентской части
└── data/ Данные, используемые сервером (файлы в формате JSON)

CMakeLists.txt
Управляет процессом сборки. Подключает библиотеку Boost, задаёт исполняемые цели и копирует директорию data/ в выходную директорию сборки.
При добавлении новых исполняемых файлов или данных необходимо обновить соответствующие секции.

common/
Содержит код, общий для клиента и сервера:
utils.hpp / utils.cpp — вспомогательные функции (HTTP, парсинг, обработка ошибок).
Функции, используемые только одной стороной, следует размещать в соответствующих модулях.

server/
Реализует серверное приложение.
main.cpp — точка входа.
server.hpp / server.cpp — базовая логика приёма подключений, распределения по пулу потоков, обработки HTTP-запросов.
handlers.hpp / handlers.cpp — бизнес-логика:
handle_get_cars() — получение списка автомобилей;
handle_admin_request() — обработка административных команд (в текущей реализации — заглушка).
Дополнительные функции (фильтрация, расчёт пошлин) следует реализовывать в этом модуле.

Сервер прослушивает порт 8080 для клиентских запросов.

Данные загружаются из data/cars.json. Поддержка новых файлов (например, tariffs.json) требует расширения логики инициализации.

client/
Консольное приложение с интерактивным меню.
main.cpp — управление вводом пользователя и вызов клиентских функций.
client.hpp / client.cpp — реализация HTTP-запросов к серверу.
В перспективе возможны:
расширенное меню (поиск, расчёт доставки);
структурированный вывод (табличный формат);
локальное кэширование.

data/
Хранит статические данные, необходимые серверу.
cars.json — база автомобилей в формате JSON.
Путь к файлам определяется относительно рабочей директории сервера (./data/). При сборке файлы автоматически копируются в build/data/.

Требования к системе
Операционная система: Linux (тестировалось на Ubuntu 22.04 / Debian 12)
Компилятор: g++ (версия 9 и выше)
Сборочная система: cmake (версия 3.16 и выше)
Зависимости: libboost-all-dev

Установка зависимостей (Ubuntu/Debian):
sudo apt update
sudo apt install build-essential cmake libboost-all-dev

Сборка проекта

Перейдите в корневую директорию проекта:
cd testCarDelivery
Создайте директорию сборки и выполните конфигурацию:
mkdir build && cd build
cmake ..
Запустите сборку:
make
После успешной сборки в директории build/ появятся:
исполняемый файл server;
исполняемый файл client;
директория data/ с содержимым из testCarDelivery/data/.

Запуск

Сервер
Выполните в терминале:
./server

Сервер ожидает входящие HTTP-запросы на порту 8080.
Для остановки сервера используйте Ctrl+C.

Клиент
В отдельном терминале:
./client

По умолчанию клиент подключается к localhost:8080.
Для удалённого подключения укажите IP-адрес сервера как аргумент:
./client <IP_адрес_сервера>

Если файл client получен от другого пользователя, перед запуском назначьте права на исполнение:
chmod +x client

Сетевые настройки
Для корректной работы в локальной сети убедитесь, что порт 8080 открыт на сервере.

Проверка состояния фаервола (если используется ufw):
sudo ufw status

Разрешение трафика:
sudo ufw allow 8080/tcp

Для временного отключения фаервола (только в тестовой среде):
sudo ufw disable
